##js是如何实现继承的？

说到如何实现继承这个问题，要先理解构造函数，原型对象和实例三者的关系，在原型和原型链中有介绍。 js实现继承方式：主要有原型链继承，构造函数继承，组合继承，寄生组合继承，es6继承。

1：原型链继承：通过修改子类原型为父类的实例，实现子类可以访问到父类构造函数以及原型上的所有属性和方法。

  function Parent(){
      this.name="dog"
  }
  Parent.prototype.getName=function(){
      return this.name;
  }

  function Child(){};
  // 这里也可以直接写出Child.prototype = Parent.prototype
  // 但是这样就不能访问到父类的构造函数的属性了，即this.name
  Child.prototype=new Parent();

  var child=new Child();
  child.getName()//dog


  缺点：父类构造函数的引用类型会被所有子类实例共享，当其中一个子类实例修改了这个值，其他子类的这个值也会修改。

2：构造函数继承 通过修改父类的构造函数的this实现的。我们在子类构造函数中执行父类构造函数，将父类构造函数this指向子类的this

  
    function Parent(){
        this.name=['dog']
    }
    function Child(){
        //子类构造函数内修改父类构造函数this指向为子类构造函数this
        Parent.call(this)；
    }

    var child =new Child()
    child.name.push('god9');
    var child2=new Child()
    console.log(child2.name)//['dog']
    优点：很好的解决了原型链继承方式父类构造函数内引用类型被所有子类实例共享的问题，通过call方法向构造函数传参。
    缺点：所有方法都需要定义在构造函数中，每次都需要重新创建。

3：组合继承 结合原型链和构造函数继承方式

  function Parent(){
      this.name='dog'
  };
  Parent.prototype.getName=function(){
      return this.name
  }

  function Child(){
      Parent.call(this)
      this.name='dog2'
  }

  Child.prototype=new Parent();
  // 需要重新设置子类的constructor，Child.prototype = new Parent()相当于子类的原型对象完全被覆盖了
  Child.prototype.constructor=Child

  优点：解决了原型链继承方式引用类型共享问题和构造函数继承方法需要多次创建问题。
  缺点：父类构造函数被调用了两次，同时子类实例和子类原型上都会出现name属性，根据原型链机制，不会访问到原型对象上的同名属性但是总之是不好的

4：寄生组合继承 其实实在组合继承上解决了构造函数被调用两次问题

 
   function Parent() {
  this.name = 'fedaily'
}

Parent.prototype.getName = function() {
  return this.name
}

function Child() {
  Parent.call(this)
  this.topic = 'fe'
}

// 仔细看这个函数的实现
inherit(Child, Parent)
//关键在于inherit的实现
function inherit(child, parent) {
    var prototype=object(parent.prototype)；
    prototype.constructor=child
    child.prototype=prototype
}
//这个函数类似复制了一份父类的原型对象，通过object方法复制了一份父类原型对象是为了解决如果直接将子类的原型对象赋值为父类原型对象的话
//，那么修改子类原型对象其实就相当于修改了父类的原型对象
function object(o){
    function F(){}
    F.protytype=o
    return new F()
}

优点：很好的解决了原型链继承引用类型子类所有实例共享问题和构造函数方法多次创建问题以及组合继承父类构造函数调用两次的问题。

5：es6继承

 
class Parent {
    constructor(){
        this.name='dog'
    }
    getName(){
        return this.name
    }
}

class Child extends Parent{
    constructor(){
        // 这里很重要，如果在this.topic = 'fe'后面调用，会导致this为undefined，具体原因可以详细了解ES6的class相关内容，这里不展开说明
        super()
       this.topic = 'fe'
    }
}

var child=new Child();
console.log(child.getName())//dog
