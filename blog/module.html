js模块化很好解决了js文件之间依赖的问题，他将一大堆js和功能块细分成细小的功能块， 使得代码维护性和可读性大大的提升

# commonjs

    介绍：commonjs规范主要用于server端的模块化规范，他是同步加载的，而且是在运行时候加载，
    有缓存的，CommonJS模块的加载机制是，引入值是抛出值的拷贝。
    也就是说，一旦输出一个值，模块内部的变化就影响不到这个值。

    使用：commonjs用法是使用modeule.exports向外抛出接口，require引入其他模块使用。
    也可以使用exports暴漏变量或者函数，他其实就是指向module.exports.commonjs不能直接被浏览器识别，
    需要通过browerfy编译为浏览器可以直接识别的语法

    commonjs是如何加载模块的？
    require命令是CommonJS规范之中，用来加载其他模块的命令。它其实不是一个全局命令，
    而是指向当前模块的module.require命令，而后者又调用Node的内部命令Module._load
    在module.load下，有几个执行步骤，
    1：判断缓存中是否有指定模块，如果有，取缓存
    2：无的话创建一个新的modeul实例，
    3：将他存入缓存中4：执行modeule.load加载指定模块内容
    5：内容加载后，再通过module.compiler执行文件代码，
    6：如果加载和编译过程出现错误，则会从缓存中删除该模块，反之则返回该模块的module.exports

# es6module

    
    使用：es6模块是目前应用最广泛的模块化之一，他也是需要通过babel编译成es5语法，
    commonjs，再通过browerfy编译为浏览器可以直接识别的语法。
    他模块向外是通过export 和import引入，es6则是在编译时候加载的，是异步加载模块，
    他是一个对值的引用，模块内部值发生变化时候外部值也会响应发生变化。
    当浏览器的js引擎静态分析时候，识别到了import命令，就会生成一个只读引用。
    等到脚本真正执行时，再根据这个只读引用，到被加载的那个模块里面去取值。

# Amd

requirejs是amd规范的产物，首先，他是依赖前置，需要将需要加载依赖提前加载好，无论你要不要用，通过define定义，return暴漏出去，通过require引用，通过回调函数形式执行其他代码，

# cmd

seajs则是阿里团队研究出来的，结合commonjs和requirejs的规范，使用上和require很相似，暴漏则是通过commonjs规范，define定义通过回调内的参数require，module，export执行，他对于依赖模块，则是就近原则，需要用到时候require进来使用即可
