# this是什么？

this 是在运行时进行绑定的，并不是在编写时绑定，它的上下文取决于函数调用时的各种条件。
this 的绑定和函数声明的位置没有任何关系，只取决于函数的调用方式

# 绑定this几种方式

 1：默认绑定
 var a=2;
 function test(){
    console.log(this.a)
 }
 test()//2
 调用函数test后，执行打印操作，此时this其实是执行函数调用者，其实是window.test()，
 所以window.a=2;


 2：隐式绑定
    function test(){
        console.log(this.a)
    }

    var obj={
        a:66,
        test:test
    }
    obj.test()//66
    是调用位置是否有上下文对象，此时函数调用者是obj对象，所有this，指向obj，从而this.a=66
    注意： 对象属性引用链中只有最顶层或者说最后一层会影响调用位置。
    function test(){
        console.log(this.a)
    }

    var obj2={
        a:66,
        test:test
    }
    var obj={
        a:55,
        obj2:obj2
    }
    obj.obj2.test()//66

    隐式丢失例子：
    function bar(){
        console.log(this.num)
    }
    var obj={
        num:1,
        bar:bar
    }
   
    var test=obj.bar()
     var num=22;
     test()//22
    虽然test是obj.bar的一个引用，但是它是引用函数本身，因应用了默认绑定。this指向全局变量。



3：显示绑定（可以通过 apply 、call 和 bind 将函数中的 this 绑定到指定对象。）
显式绑定的核心是 JavaScript 内置的 call(..) 和 apply(..) 方法，
这两个方法在 JavaScript 提供的绝大多数函数以及开发者自己创建的所有函数上都可以使用。

注:如果call()和apply()的第一个参数是null，在非严格模式下，
第一个参数为null或者undefined时会自动替换为指向全局对象(window)，


4：强绑定
其实也是显示绑定的一种，不过很好的解决了隐式丢失问题

5：new绑定
只是被 new 操作符调用的普通函数而已。


优先级：new绑定 > 显示绑定 > 隐式绑定 > 默认绑定，
所以es6的箭头函数this指向他外层函数的this指向，
且无法修改

改变this指向的call，apply，bind

call和apply类似，第一个参数都是指向this的绑定函数，第二个参数不同，call传入是单个参数形式， apply传入是数组参数形式。call和apply可以用作，改变this指向，继承（调用他人函数），调用函数 bind（低版本的 IE 中不兼容）绑定，bind 方法不会立即执行，而是返回一个改变了上下文this后的函数。 而原函数func中的 this并没有被改变，依旧指向全局对象 window。

bind: 1:创建一个新的函数。 2：bind被调用时，新函数的this被bind的第一个参数指定 bind手写polyfill：

if(!function.prototype.bind){
    function.prototype.bind=function(oThis){
        if(typeof this!='function'){
            throw new TypeError('当前调用bind方法的不是函数！')
        }

        var args=Array.prototype.slice.call(arguments,1)
        var fToBlod=this;
        var fBound=function(){
            return fToBlod.apply(oThis,args.concat(Array.prototype.slice.call(arguments)))
        }
        return fBound;
    }
}
