# 对babel的认知

babel是js的编译器，可以将es6+的代码转换为向后兼容的js语法，从而使低版本和当前浏览器都可以识别他。

1：可以对js代码进行转换
2：可以通过@babel/polyfill模块，用polyfill方式为当前环境补充没有的特性
3：还可以进行源码转换

# 对babel的编译原理（三个过程）

1：解析：将代码字符串解析成抽象语法树（AST）
    大致分为两个过程：1：进行分词就是将语法字符串进行循环遍历拆成最小单元数组.
                    2：语法分析，建立每个最小单元之间的语法关系.


2：转换：将抽象语法树进行转换操作
这个阶段就会使用到插件了，不使用的话原本样子输出源码.

了解plugin和preset区别
    1：plugin则是插件意思，转换过程中使用到了引入即可实现转换
    2：preset是babel官方提供的插件集，每年更新一次只更新本年批准的内容，（babel-preset-env是最全的插件集合）
同时存在执行顺序：
Plugin 会运行在 Preset 之前。
Plugin 会从第一个开始顺序执行。
Preset 的顺序则刚好相反(从最后一个逆序执行)。


3：生成：将转换后的代码再生成代码字符串（通过babel-genenrotor将ast树转换为es5浏览器可以识别的代码）
